# Thesis Rewrite Guidelines

## Purpose

These guidelines define how the kickoff document should be transformed into thesis chapters.

The primary objective is to preserve the original kickoff text as much as possible. The task is not to rewrite the thesis from scratch, but to convert the kickoff document into thesis-ready chapters with only the minimum necessary changes.

## Core Principle

The highest-priority rule is:

**Preserve the original kickoff wording to the maximum possible extent.**

This means:

- Reuse existing sentences whenever they are already logically sound.
- Reuse the original paragraph order whenever it already works.
- Reuse the original citation logic and literature positioning whenever possible.
- Avoid unnecessary paraphrasing, expansion, compression, or stylistic rewriting.

## What Must Be Preserved

The following should be preserved by default unless there is a strong reason to change them:

- The main research narrative from the kickoff document.
- The original paragraph structure.
- The original argument sequence.
- The original research questions.
- The original contribution logic.
- The original citation locations and citation intent.

## What May Be Changed

Changes are allowed only when they are necessary for thesis consistency.

### 1. Proposal-to-thesis wording

Convert proposal phrasing into thesis phrasing when needed.

Examples:

- `this research will explore` -> `this thesis examines`
- `this study proposes to develop` -> `this thesis develops`
- `the following chapters will` -> `the remainder of the thesis is organized as follows`

### 2. Terminology alignment

When referring to the upstream model previously associated with Sara, do **not** use the word `tactical`.

Use one of the following instead:

- `upstream OR model`
- `rebalancing model`
- `interval-based OR model`
- `relocation guidance from the OR model`

Do **not** use the expression:

- `Sara-aligned`

If a neutral environment description is needed, use expressions such as:

- `data-driven simulation environment`
- `simulation environment calibrated to the case setting`
- `externally injected relocation guidance`

### 3. Thesis structure alignment

Adjust only the chapter-structure paragraph when necessary so that it matches the actual thesis template and chapter organization.

### 4. Project-state consistency

Revise only the parts that would otherwise conflict with the actual project status.

Examples of acceptable alignment:

- The thesis does not redesign the upstream OR model.
- The thesis focuses on trip-level selective execution.
- The simulation environment has already been established as the experimental testbed.
- Reinforcement-learning-based evaluation may be described as part of the methodological framework if full final experiments are not yet completed.

## What Should Not Be Done

The following are not allowed unless explicitly requested:

- Rewriting chapters from scratch.
- Reordering paragraphs without necessity.
- Replacing the original kickoff argument with a new one.
- Adding large amounts of new text just to sound more academic.
- Removing existing citations simply for stylistic simplification.
- Introducing terminology that conflicts with the agreed vocabulary.

## Citation Rules

Introduction chapters in the thesis should still contain citations.

However, the citation logic in Chapter 1 should remain problem-oriented rather than fully review-oriented.

Citations in Chapter 1 should support:

- the importance of shared micromobility,
- the service-level and battery-availability problem,
- the temporal resolution gap,
- the intervention precision gap,
- the motivation for incentive-based execution refinement.

Systematic literature comparison belongs primarily in Chapter 2.

Whenever possible, keep the same citation logic already used in the kickoff document.

## Writing Style Rules

- Keep academic English clear and direct.
- Prefer preserving the original sentence structure over stylistic polishing.
- Avoid over-editing for elegance if the original sentence is already usable.
- Do not exaggerate results that are not yet fully established.
- Do not present the thesis as if it were redesigning the entire upstream planning model.

## Fixed Research Positioning

The thesis should consistently be framed as studying:

- `trip-level selective execution`
- `real-time refinement layer`
- `selective implementation of interval-based relocation guidance`

The thesis should not be framed as:

- a full redesign of the upstream OR model,
- an end-to-end replacement of the planning layer,
- a complete redefinition of the rebalancing problem.

## Operating Rule for Future Chapter Writing

For every future chapter, the default process is:

1. Start from the kickoff wording.
2. Preserve as much original text as possible.
3. Apply only the minimum required thesis-oriented edits.
4. Keep terminology consistent with this document.
5. Keep the original literature logic whenever possible.

## Priority Order

When rules conflict, use this priority order:

1. Preserve kickoff wording as much as possible.
2. Maintain thesis consistency.
3. Maintain agreed terminology.
4. Improve clarity only when necessary.
5. Add new content only when explicitly requested or strictly required.
